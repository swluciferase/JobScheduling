# OrderFlow Phase 1: Order MVP with Lot/UDI Traceability + DHR (v0.2)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver a working, QMS-grade order tracking MVP — customers, orders with Lot/UDI/Serial traceability, the 11-stage manufacturing template (incoming inspection → in-process QC → final QC → release for shipment), document storage, auto-compiled DHR (per 21 CFR 820.184), SLA alerts, multi-channel notifications, logistics integration, and kanban + report UIs — so internal sales/ops/QA can manage a real Class II order from receipt to delivery with full audit trail and electronic signatures on every QMS-critical step.

**Architecture:** Builds on Phase 0 foundation. Adds business tables (customers, orders, order_items with Lot/UDI/Serial, order_stages, stage_templates, acceptance_records, documents, alerts, notifications, logistics_events). All writes go through `db` helper which automatically wraps audit (no `tenant_id` filtering needed — single-tenant). Stage progression is a state machine driven by per-organization JSON templates with embedded `requires_esig` / `requires_acceptance` / `qms_clause` metadata. Critical stage advances require an active reauth_token (Phase 0 framework) and write paired `e_signatures` rows. DHR PDF assembled on demand from `acceptance_records` + `e_signatures` + `order_items` + `documents`. Alerts run as scheduled Worker (cron) writing into a Cloudflare Queue, consumed by a notifications dispatcher with Email/Teams/Slack adapters.

**Tech Stack:** Same as Phase 0 + Resend (email), Microsoft Graph (Teams channel webhook), Slack Web API, `pdf-lib` (DHR + system-generated docs, Workers-compatible), R2 (document storage + immutable bucket for DHR final), Cloudflare Queues, Cloudflare Cron Triggers, 17track REST API, dnd-kit (kanban DnD).

**Phase boundary:** Ends when a sales user can: create customer → create order with items (Lot/UDI populated by ops at incoming inspection) → see order on kanban → ops advances through stages with QC e-signatures → QA manager signs Release for Shipment → DHR PDF auto-generated and stored in R2 immutable bucket → user receives Email+Teams alert when stage breaches SLA → views order's logistics tracking → runs basic monthly report. Mobile/PWA, barcode UDI scanning, accountant payment screens, NCR/CAPA, MCP, and Annual Audit deferred to Phase 2/3/4.

  target TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('queued','sending','sent','failed','dropped')),
  sent_at TEXT,
  error TEXT,

  created_at TEXT NOT NULL
);

  carrier TEXT NOT NULL,
  tracking_no TEXT NOT NULL,

  event_desc TEXT,
  event_time_utc TEXT NOT NULL,
  location TEXT,
  raw_json TEXT,
  created_at TEXT NOT NULL,
  UNIQUE(order_id, event_code, event_time_utc)
