# OrderFlow Phase 1: MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver a working order tracking MVP — customers, orders, stage progression with templates, document storage, SLA alerts, multi-channel notifications, logistics integration, kanban + report UIs — so internal sales/ops can manage a real order from receipt to delivery.

**Architecture:** Builds on Phase 0 foundation. Adds business tables (customers, orders, order_items, order_stages, stage_templates, documents, alerts, notifications, logistics_events). All queries go through `getDb(d1).where(eq(table.tenant_id, ctx.tenantId))`. Stage progression is a state machine driven by per-tenant JSON templates. Alerts run as scheduled Worker (cron) writing into a Cloudflare Queue, consumed by a notifications dispatcher with Email/Teams/Slack adapters.

**Tech Stack:** Same as Phase 0 + Resend (email), Microsoft Graph (Teams), Slack Web API, jsPDF (system-generated docs), R2 (document storage), Cloudflare Queues, Cloudflare Cron Triggers, 17track REST API, dnd-kit (kanban DnD).

**Phase boundary:** Ends when a sales user can: create a customer → create an order with items → see the order on a kanban → advance through stages → upload/generate documents → receive an Email+Teams alert when a stage breaches SLA → view a finished order's logistics tracking → run a basic monthly report. Mobile/PWA, barcode, accountant payment screens, MCP, and self-service signup are deferred to Phase 2/3.

```ts
export async function instantiateStages(db, order, templateId) {
  const tpl = await db.query.stageTemplates.findFirst({ where: and(eq(stageTemplates.id, templateId), eq(stageTemplates.tenantId, order.tenant_id)) });
  if (!tpl) throw new Error('template not found');
  const def = StageTemplateSchema.parse(JSON.parse(tpl.stagesJson));
  const rows = def.stages
    .filter(s => !s.conditional || evalCondition(s.conditional, order))
    .map((s, i) => ({
      id: ulid(), tenantId: order.tenant_id, orderId: order.id,
      stageKey: s.stage_key, stageName: s.name_zh, sequence: i+1,
      status: i===0 ? 'in_progress' : 'pending',
      slaHours: s.sla_hours ?? null,
      hiddenInProgressbar: s.hidden_in_progressbar,
      startedAt: i===0 ? new Date() : null,
    }));
  await db.insert(orderStages).values(rows);
  return rows;
}
function evalCondition(expr, order) {
  // simple: "order.requires_approval == true" / "order.total_amount > 500000"
  const ctx = { order };
  return new Function('ctx', `with(ctx){return ${expr}}`)(ctx);
}
```
